# E-Ink Home Dashboard

Local FastAPI renderer for an 800x480 Waveshare 7.5 inch e-paper dashboard. It fetches Home Assistant state data, renders a black-and-white PNG with Pillow, caches renders for 30 seconds, and saves the latest image as `dashboard.png`.

No Docker is required.

## Repository Contents

- `app.py` renders the dashboard and exposes the HTTP endpoints.
- `esphome/esp32-s3-dashboard.example.yaml` downloads the rendered image and displays it on a Waveshare e-paper panel.
- `esphome/secrets.example.yaml` documents required ESPHome secrets without publishing real credentials.
- `eink-renderer.service` is a systemd template installed with paths and user values generated by `make service-install`.

## Network Roles

There are two different addresses:

- `HA_URL` is the Home Assistant API URL, for example `http://<home-assistant-ip>:8123`.
- The renderer is this FastAPI app, for example `http://<renderer-ip>:8080`.

ESPHome should fetch the renderer URL:

```text
http://<renderer-ip>:8080/dashboard.png
```

## Configure Home Assistant

Copy the example environment file and edit it:

```bash
cp .env.example .env
nano .env
```

Required values:

```dotenv
HA_URL=http://<home-assistant-ip>:8123
HA_TOKEN=your_home_assistant_long_lived_access_token
WEATHER_ENTITY=weather.yandex_weather
DEBUG_MODE=false
```

The app uses `HA_URL` and `HA_TOKEN` only for Home Assistant API calls.
`WEATHER_ENTITY` defaults to `weather.yandex_weather`.
`DEBUG_MODE=false` keeps diagnostic text off the final e-paper image.

Fetched Home Assistant entities:

- `calendar.family`
- `sensor.temp_living_temperature`
- `sensor.temp_living_humidity`
- `sensor.temp_kitchen_temperature`
- `sensor.temp_kitchen_humidity`
- `sensor.open_exchange_rates_usd_rub`
- `sensor.kanye_quote`
- `weather.yandex_weather`

## Run Locally

```bash
make run
```

Manual equivalent:

```bash
uvicorn app:app --host 0.0.0.0 --port 8080
```

Endpoints:

- `GET /dashboard.png` renders or returns the cached 800x480 PNG.
- `GET /preview` shows the PNG centered on a dark page.
- `GET /health` returns renderer status JSON.

Dashboard layout:

- Top metadata line with `HOME` and Moscow time update.
- Main family calendar event as the hero, with up to 3 compact upcoming events from `GET /api/calendars/calendar.family`.
- Current weather and 3-day hourly forecast summary on the right.
- Living room, kitchen, and smaller USD/RUB metrics below.
- Tiny one-line quote footer.

Open from another local machine:

```text
http://<renderer-ip>:8080/preview
http://<renderer-ip>:8080/dashboard.png
http://<renderer-ip>:8080/health
```

## systemd Background Service

Install, enable on boot, and start:

```bash
make service-install
```

This may prompt for `sudo` only when installing or controlling the systemd
unit; the virtual environment is created as the current user.

Manage it:

```bash
make service-status
make logs
make service-restart
make service-stop
```

The installed unit runs the equivalent of:

```text
<project-directory>/.venv/bin/uvicorn app:app --host 0.0.0.0 --port 8080
```

`make service-install` fills the project directory and current user/group into
the service template. Re-run it if the clone location or service account
changes.

## Update Code

After pulling or changing code:

```bash
make update
```

This runs `git pull --ff-only` when the directory is a valid git repo, reinstalls dependencies, and restarts the service if it is installed.

If this directory is not a valid git repo, `make update` skips `git pull` and still updates dependencies/restarts the service.

## ESPHome Device Configuration

The complete ESP32-S3 example is in
[`esphome/esp32-s3-dashboard.example.yaml`](esphome/esp32-s3-dashboard.example.yaml).
It downloads `dashboard.png`, displays it on a Waveshare 7.50inV2 panel, exposes a
manual refresh button, and includes the onboard RGB LED configuration.

Set it up without committing credentials:

```bash
cp esphome/secrets.example.yaml esphome/secrets.yaml
```

Edit `esphome/secrets.yaml` with Wi-Fi credentials, newly generated ESPHome API
and OTA values, and the renderer URL. `esphome/secrets.yaml` is ignored by Git.

Pin assignments and the exact Waveshare model value should match your ESPHome
device. The example uses plain HTTP for a trusted local network; do not expose
the renderer endpoint directly to the internet.

## Publication Safety

The repository should include `.env.example` and `esphome/secrets.example.yaml`
only. Never commit `.env`, `esphome/secrets.yaml`, Home Assistant long-lived
tokens, Wi-Fi passwords, ESPHome API encryption keys, or OTA passwords.
